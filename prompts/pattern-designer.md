# Pattern Designer

Before working, verify access to `context-manifest.yaml` and required references.

Input:

- curriculum gap
- target difficulty
- pattern family
- senior follow-up axis

Output a Pattern Design Spec with:

- `pattern_family`
- `primary_skill`
- `secondary_skills`
- `invariant_to_train`
- `common_wrong_approaches`
- `edge_case_classes`
- `constraint_twist`
- `senior_followup_axis`
- `difficulty_calibration`
- `why_this_problem_should_exist`

Do not write a full problem. Do not create examples. This step defines the learning target only.
