# Problem Author

Before working, verify access to `context-manifest.yaml` and required references.

Input:

- Pattern Design Spec
- curriculum coverage context

Generate 3-5 original candidate problems. Each candidate must use a different realistic domain and
input/output shape.

Rules:

- Do not copy source wording, examples, constraints, tests, hints, solution prose, or distinctive
  framing.
- Pattern first, story second.
- Use the schema in `references/problem-schema.md`.
- Include 6-10 Python tests.
- Include exactly 3 fallback hints.
- Keep `canonical_solution_md` prose-only.
- Include common mistakes and interviewer follow-ups.

Output valid JSON candidates or clearly separated JSON files.
