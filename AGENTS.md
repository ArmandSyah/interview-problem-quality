# Agent instructions

This repo defines reusable quality rules for coding interview problem generation
and review.

Rules:

1. Read `context-manifest.yaml` first.
2. Load task-specific references from `references/`.
3. Keep `interview-mcp` runtime-only.
4. Put private drafts and review outputs in `interview-mcp-ops`.
5. Run deterministic validation before asking for human review.
6. Do not copy wording, examples, solutions, distinctive framing, or code from sources.
