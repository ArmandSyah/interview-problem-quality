# Source use policy

Sources such as CtCI, EPI, CLRS, LeetCode discussions, academic textbooks, blog posts, and
YouTube transcripts may be used only for:

- Pattern discovery.
- Difficulty calibration.
- Common mistake discovery.
- Constraint inspiration.
- Follow-up inspiration.

Never copy:

- Problem wording.
- Examples.
- Solution prose.
- Code.
- Distinctive story framing.
- Distinctive input/output shape when that shape makes the problem recognizable.

When adapting an idea, change:

- Domain.
- Names.
- Input/output shape.
- Constraints.
- Examples.
- Tests.
- Explanation.
- Follow-ups.

Source notes should summarize abstract learning targets, not reproduce source text. If a reviewer
can identify a known problem from the story, examples, or exact constraints, treat that as at least a
major originality issue.
