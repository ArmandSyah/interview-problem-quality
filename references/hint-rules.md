# Hint rules

Every accepted problem must include exactly three fallback hints.

Depth expectations:

- Depth 1: conceptual nudge only. It may name the broad direction but should not name the full
  algorithm if that gives away the problem.
- Depth 2: may name the pattern, data structure, invariant, or state being maintained.
- Depth 3: may describe the full algorithm in natural language.

Forbidden in all hints:

- Solution code.
- Function bodies.
- Line-by-line pseudocode.
- Code fences.
- Direct variable-by-variable implementation instructions.
- Language that says exactly what to type next.

Good hints preserve interview value. They should help the candidate recover the next idea while
still requiring them to produce the implementation and explain the invariant.
