# Canonical solution rules

`canonical_solution_md` explains the intended reasoning path. It is not a code solution.

It should include:

- The core observation.
- The invariant or state being maintained.
- Why the algorithm is correct.
- Time and space complexity.
- Important edge cases.
- Trade-offs against at least one plausible weaker approach.

It must not include:

- Full code.
- Function bodies.
- Long pseudocode.
- Code fences.
- Copyable implementation steps disguised as prose.

The goal is to support reviewers and future hint generation without leaking a ready-to-submit
answer into agent context.
